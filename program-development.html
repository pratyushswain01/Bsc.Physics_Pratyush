Unit 2: Programming Fundamentals & Logic

Section A (2 Marks Questions)

1. What is an algorithm? Write down characteristics of an algorithm.

An algorithm is a well-defined, step-by-step procedure for solving a problem. Its characteristics are: Finiteness, Definiteness, Input, Output, and Effectiveness.

2. What is Pseudocode? Give one example of it.

Pseudocode is an informal, high-level description of an algorithm, using natural language conventions. Example (to add two numbers):

pseudocode
START
  READ A, B
  SUM = A + B
  PRINT SUM
STOP

3. What is a flow chart? Give the name of different symbols used in flow chart.

A flowchart is a graphical representation of an algorithm. Common symbols include: Oval (Start/End), Parallelogram (Input/Output), Rectangle (Process), and Diamond (Decision).

4. What is top-down design approach?

Top-down design is a problem-solving approach where a complex problem is broken down into smaller, more manageable sub-problems. This process is repeated until each sub-problem is simple enough to be solved directly. It is characteristic of procedural programming.

5. What is syntax error? How is it different from semantic error?

A syntax error violates the grammatical rules of a language (e.g., a missing semicolon). A semantic error is syntactically correct but logically meaningless (e.g., `x * y = z;`). Syntax errors are caught by the compiler; semantic errors may or may not be.

7. Differentiate between compile time and run-time error.

A compile-time error (like a syntax error) is detected by the compiler *before* the program is executed. A run-time error occurs *during* program execution, often due to an illegal operation like division by zero, even if the syntax is correct.

9. What is an assembler?

An assembler is a translator that converts a program written in assembly language (which uses mnemonics like ADD, MOV) into machine language (binary code).

10. Differentiate between algorithm and flow chart.

An algorithm is a textual, step-by-step description of a solution. A flowchart is a graphical, diagrammatic representation of that same solution, using symbols to show the flow of logic.

11. What is structured programming?

Structured programming is a paradigm that uses a top-down approach and relies on three basic control structures (sequence, selection, and iteration) to write clear, manageable, and modular code. It discourages the use of 'goto' statements.

12. What is a loader in programming language?

A loader is a part of the operating system that takes an executable file from secondary storage and loads it into main memory (RAM) to prepare it for execution by the CPU.

13. What is linker?

A linker is a utility program that combines one or more object files (generated by a compiler) and library files into a single executable file.

14. Define testing.

Testing is the process of executing a program with the intent of finding errors. It is a quality control activity to verify that the software meets the specified requirements and is free of defects.

15. What is debugging?

Debugging is the process of locating, analyzing, and fixing errors or bugs that have been identified in a computer program, often after they have been found during testing.

16. What is link time error?

A link-time error occurs when the linker is unable to resolve a reference, such as when a function is called but its definition cannot be found in any of the linked object files or libraries.

17. What are the characteristics of a good program?

A good program should be: Correct and Reliable, Efficient (in time and space), Readable, Maintainable, and Portable.

18. What is bottom-up design approach?

Bottom-up design is a problem-solving approach where individual components or modules are designed and built first. These modules are then integrated to form larger systems. It is characteristic of object-oriented programming.

19. What are translators?

Translators are system programs that convert source code written in one programming language into another, typically machine language. The main types are compilers, interpreters, and assemblers.

20. What are control structures?

Control structures are programming constructs that control the flow of execution. The three basic types are: Sequential (step-by-step), Selection (if-else), and Iteration (loops like for/while).

Section B (5 Marks Questions)

1. What is flow chart? Explain different symbols used in a flow chart.

A flowchart is a graphical or pictorial representation of an algorithm. It uses standard symbols to depict the sequence of operations and the flow of control in a program, making the logic easier to visualize and understand.

Common Flowchart Symbols:
  • Oval (Terminal): Represents the Start or End point of the algorithm.
  • Parallelogram (Input/Output): Represents an operation where data is read from the user (Input) or displayed to the user (Output).
  • Rectangle (Process): Represents a computational step, such as an assignment or a calculation.
  • Diamond (Decision): Represents a point where a decision is made. It has one entry point and at least two exit points (e.g., Yes/No, True/False).
  • Arrows (Flow lines): Indicate the direction of flow and connect the symbols.

3. Explain different languages translators in language processor.

Language translators are system programs that convert source code from one language to another. The main types are:

  • Assembler: Translates assembly language (using mnemonics) into machine language (binary code). It performs a one-to-one conversion.
  • Compiler: Translates an entire high-level language program into machine code at once. It generates an executable file and reports all syntax errors together after scanning the whole program.
  • Interpreter: Translates and executes a high-level language program line by line. It does not create an executable file and stops at the first error it encounters.

4. Write down short notes on assembler, compiler and interpreter.

Assembler: An assembler is a translator for assembly language. It converts symbolic instruction codes (mnemonics) like `ADD`, `MOV`, and `SUB` into their binary equivalents that the CPU can execute. This translation is direct and fast, as each assembly instruction typically corresponds to one machine instruction.

Compiler: A compiler is a translator for high-level languages like C++ and Java. It works by taking the entire source code file, checking it for syntax errors, and if none are found, translating it into a complete, standalone executable file. This process is done before the program is run, which allows the final executable to run very quickly.

Interpreter: An interpreter also works with high-level languages like Python and JavaScript. Instead of translating the whole program beforehand, it reads one line of the source code, translates it, executes it, and then moves to the next line. This makes debugging easier but generally results in slower overall execution compared to a compiled program.

6. What is an error? Explain different types of errors in programming language.

An error (or bug) is a mistake in a program that causes it to behave in an unintended way. The main types are:

  1. Compile-time Errors (Syntax Errors): These errors violate the grammatical rules of the language and are caught by the compiler before the program runs. Examples include missing semicolons or misspelled keywords.
  2. Run-time Errors: These errors occur during program execution. The code is syntactically correct, but an illegal operation is attempted, such as division by zero or trying to access a file that doesn't exist.
  3. Logical Errors: These are the most difficult to find. The program compiles and runs without crashing, but it produces the wrong output because of a flaw in the programmer's logic (e.g., using `+` instead of `-` in a formula).

7. Differentiate between object-oriented and procedure-oriented programming.

Procedure-Oriented Programming (POP) focuses on functions or procedures. The program is divided into functions, and data is often global and can be accessed freely. Example: C.

Object-Oriented Programming (OOP) focuses on objects, which bundle data and the functions that operate on that data together. It emphasizes data security through concepts like encapsulation. Example: Java, C++.

8. What is programming language? Explain different generations of programming language.

A programming language is a formal language with a set of rules and syntax used to write instructions for a computer. Its generations are:

  • 1GL (First Generation): Machine Language (binary).
  • 2GL (Second Generation): Assembly Language (mnemonics).
  • 3GL (Third Generation): High-Level Languages (e.g., C, Java, Python).
  • 4GL (Fourth Generation): Very High-Level Languages, often for specific purposes (e.g., SQL for databases).
  • 5GL (Fifth Generation): Languages for AI and logic programming (e.g., Prolog).

9. Explain different steps involved in development programs.

The development of a program typically follows the Software Development Life Cycle (SDLC):

  1. Requirement Analysis: Understanding and defining the problem and what the software needs to do.
  2. Design: Planning the software's architecture, creating algorithms, and designing the user interface.
  3. Implementation (Coding): Writing the source code in a chosen programming language based on the design.
  4. Testing: Executing the program to find and identify bugs or defects.
  5. Deployment: Releasing the software to users.
  6. Maintenance: Updating and fixing the software after its release to handle new requirements or discovered bugs.

10. Explain programming paradigm and design in programming language.

A programming paradigm is a fundamental style or approach to programming. It dictates how programs are structured and how problems are solved. Major paradigms include Imperative (Procedural, Object-Oriented) and Declarative (Functional, Logic).

Programming design is the process of planning the program's structure before coding. This includes choosing the right algorithms and data structures. The design approach is often influenced by the paradigm. For instance, procedural programming uses top-down design, while object-oriented programming often uses bottom-up design.

Section C (8 Marks Questions)

1. What is flow chart? Explain standard notation of flow chart? Give one example of it.

A flowchart is a diagrammatic representation of an algorithm or a workflow. It provides a visual and easy-to-understand model of a process by using standard symbols connected by arrows to show the sequence of operations and flow of control.

Standard Notation (Symbols):
  • Oval (Terminal): Represents the Start and End points of the algorithm. Every flowchart must begin with a "Start" terminal and end with an "End" or "Stop" terminal.
  • Parallelogram (Input/Output): Used to represent any operation that involves taking input from a user or displaying output.
  • Rectangle (Process): Represents a processing step, such as a calculation (e.g., `sum = a + b`) or data manipulation.
  • Diamond (Decision): Represents a conditional statement where a decision must be made. It has one entry point and two or more exit points labeled with the possible outcomes (e.g., True/False, Yes/No).
  • Arrows (Flow Lines): Connect the symbols and indicate the direction of logical flow through the algorithm.
  • Circle (Connector): Used to connect different parts of a flowchart, especially when it spans multiple pages or is too complex to be drawn with direct lines.
Example: Flowchart to find the larger of two numbers
  1. Start with an Oval labeled "Start".
  2. Draw an arrow to a Parallelogram labeled "Input A, B".
  3. Draw an arrow to a Diamond labeled "Is A > B?".
  4. From the "Yes" exit of the diamond, draw an arrow to a Parallelogram labeled "Print A".
  5. From the "No" exit of the diamond, draw an arrow to a Parallelogram labeled "Print B".
  6. Draw arrows from both "Print" parallelograms to a final Oval labeled "End".

2. Explain briefly about control structure.

Control structures are fundamental building blocks in programming that allow a programmer to alter the normal sequential flow of execution. They enable programs to make decisions and perform repetitive tasks. Without them, a program would execute line by line from top to bottom without variation. There are three main types:

  1. Sequential Structure: This is the default control structure. Instructions are executed one after another in the order they are written in the source code.
  2. Selection (Conditional) Structure: This structure allows the program to choose between two or more alternative paths of execution based on a condition.
    • `if` statement: Executes a block of code only if a specified condition is true.
    • `if-else` statement: Executes one block of code if the condition is true, and another block if it is false.
    • `if-elif-else` (or `switch`/`case`): Allows for checking multiple conditions and executing a specific block of code for the first condition that evaluates to true.
  3. Iteration (Looping) Structure: This structure allows a block of code to be executed repeatedly as long as a certain condition is met.
    • `while` loop: A pre-test loop that repeatedly executes a block of code as long as its condition remains true. The condition is checked *before* each iteration.
    • `do-while` loop: A post-test loop that executes a block of code at least once, and then continues to repeat as long as its condition remains true. The condition is checked *after* each iteration.
    • `for` loop: A counting loop that is ideal for iterating a specific number of times or over the elements of a sequence (like a list or an array).

3. Write down the short notes on: (1) Loader (2) Linker (3) Testing (4) Debugging.

(1) Loader: A loader is a vital system program that is part of the operating system. Its primary job is to load an executable file from secondary storage (like a hard disk) into main memory (RAM) so it can be executed. The loader allocates memory space for the program, copies the program's code and data into that space, and then transfers control to the program's starting point to begin execution.

(2) Linker: A linker is a utility program that takes the object files generated by a compiler and combines them into a single executable file. Its main task is "symbol resolution," which means it finds the definitions for functions and variables that are referenced in one file but defined in another (such as a library file like `printf`) and links them together.

(3) Testing: Testing is a proactive process in software development aimed at finding defects. It involves executing a program with a set of test cases and comparing the actual results with the expected results. The goal is not to prove the program is correct, but to find any errors and ensure the software meets quality standards and user requirements before it is released.

(4) Debugging: Debugging is a reactive process that begins after a bug has been identified (often through testing). It is the systematic process of finding the root cause of the error, analyzing why it occurred, and then fixing the code to eliminate the bug. Debugging often involves using tools called debuggers to step through the code line by line and inspect the state of variables.

5. What is an editor? Explain different types of editor with example.

An editor is a software program that allows a user to create and modify text files. In the context of programming, these are called text editors or code editors, and they are the primary tool for writing source code.

Types of Editors:
  1. Simple Text Editors: These provide basic text editing functionalities without any special features for programming. They are lightweight and fast.
    • Examples: Notepad (Windows), TextEdit (macOS), Gedit (Linux).
  2. Code Editors (Programming Text Editors): These are advanced text editors specifically designed for writing code. They include features that make programming easier, such as:
    • Syntax Highlighting: Displays code in different colors and fonts according to its category (keywords, strings, comments).
    • Code Completion: Suggests code snippets and variable names as you type.
    • Error Highlighting: Underlines potential syntax errors in real-time.
    • Examples: Visual Studio Code (VS Code), Sublime Text, Atom.
  3. Integrated Development Environments (IDEs): An IDE is a comprehensive software suite that bundles all the essential tools for software development into a single application. It typically includes a code editor, a compiler/interpreter, and a debugger.
    • Features: They include all the features of a code editor plus tools for building, running, and debugging programs, as well as project management and version control integration.
    • Examples: PyCharm (for Python), Eclipse (for Java), Visual Studio (for .NET/C++).
