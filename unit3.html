Chapter 1: Introduction to Python

Python is a high-level, interpreted language created by Guido van Rossum. It's known for its clean syntax, making it easy to learn and powerful enough for complex applications, from web development to artificial intelligence. Its development is now managed by the Python Software Foundation (PSF).

The `print()` Function and String Formatting

The `print()` function is your primary tool for displaying output. It has helpful parameters like `sep` (separator between items) and `end` (character at the end of the line).

python
name = "Pratyush"
age = 20

# Using an f-string (modern and recommended)
print(f"My name is {name} and my age is {age}.")

# Using the older .format() method
print('My number is: {one}, and my name is: {two}'.format(one=age, two=name))

# Using custom separators
print("hello", 23.34, 123, sep='--') # hello--23.34--123

Chapter 2: Python Fundamentals

Tokens and Literals

Tokens are the smallest units of a program (keywords, variables, etc.). Literals are fixed data values like `100` (integer), `"hello"` (string), `True` (boolean), or `None`.

Variables and Data Types

A variable is a name that points to a data object. Python is dynamically typed, meaning the type is determined at runtime. You can check any variable's type with the `type()` function.

python
# Assign multiple values at once
x, y, z = "Orange", "Banana", "Cherry"

# Unpack a list into variables
fruits = ["apple", "banana", "cherry"]
a, b, c = fruits
print(a) # Prints "apple"

Type Casting

You can convert data between types explicitly.

python
data = "99"  # This is a string
str_to_int = int(data) # Now it's an integer 99
int_to_float = float(str_to_int) # Now it's a float 99.0

String Methods for Changing Case

Python strings have many useful methods to manipulate their case.

python
name="pytHON using JUPyter"
print(name.upper())        # PYTHON USING JUPYTER
print(name.lower())        # python using jupyter
print(name.title())        # Python Using Jupyter (Capitalizes each word)
print(name.capitalize())  # Python using jupyter (Capitalizes only the first word)
print(name.casefold())    # An aggressive version of lower() for all languages

Chapter 3: Mastering Operators

Operators perform actions on variables and values. You've seen Arithmetic operators (+, -, *, /, //, %, **). Others include:

  • Assignment: =, +=, -=, etc. for assigning values.
  • Comparison: ==, !=, >, <, >=, <= for comparing values.
  • Logical: and, or, not for combining boolean expressions.
  • Identity: is, is not for checking if two variables are the *same object* in memory.
  • Membership: in, not in for checking if a value exists within a sequence.
  • Bitwise: &, |, ^, ~, <<, >> for working with data at the binary level.
`is` vs `==` is a critical distinction: `==` checks if the *values* are equal. `is` checks if they are the *exact same object* in memory.

Chapter 4: Control Flow

Conditional Statements

These allow your program to make decisions. Use if for the main condition, elif for other conditions, and else as a final fallback.

Loops: for and while

Loops repeat code. The `for` loop is ideal for iterating over a sequence (like a list or a range). The `while` loop continues as long as a condition is true.

python
# A loop to find a specific number and then stop.
my_roll_numbers = [1,5,9,10,11]

for roll in range(1, 20):
    if roll in my_roll_numbers:
        print(f"Found matching roll number: {roll}")
    
    if roll > 12:
        print("Stopping search.")
        break # Exit the loop

Chapter 5: Mastering Functions

Functions bundle code into reusable, named blocks. They promote modularity and reusability.

Variable Length Arguments

`*args` and `**kwargs` are powerful tools for flexibility.
  • *args collects any extra positional arguments into a **tuple**.
  • **kwargs` collects any extra keyword arguments into a **dictionary**.
python
def my_profile(name, **data):
    print(f"Name: {name}")
    for key, value in data.items():
        print(f"- {key.title()}: {value}")

my_profile("Pratyush Swain", age=20, add="Berhampur", phno=9438721088)

Scope: Local and Global Variables

Local variables exist only inside the function they are created in. Global variables exist at the top level of the script. To **modify** a global variable from inside a function, you must declare it first with the `global` keyword.

Generators and the `yield` Keyword

A **generator** function uses `yield` instead of `return`. It produces a sequence of values over time, saving memory by not creating the whole list at once. This is what makes generators so powerful for large datasets.

Chapter 6: The World of Classes

A **Class** is a blueprint (e.g., `Car`). An **Object** is a specific instance created from that blueprint (e.g., `my_tesla`). Classes bundle data (attributes) and behavior (methods) together.

The `__init__` Constructor and `self`

The `__init__()` method is a special function called a **constructor**. It runs automatically whenever you create a new object. Its first parameter is always `self`, which represents the instance of the object itself. You use `self` to create and access the object's unique attributes.

python
class Item:
    def __init__(self, item_no, price):
        self.ino = item_no
        self.pr = price

    def calculate(self, qty):
        total = self.pr * qty
        print(f"Total price is: {total}")

# Create an object (instance)
ob = Item(101, 20)
ob.calculate(40)  # Prints "Total price is: 800"

# Modifying the object's attributes
ob.ino = 202
ob.pr = 400
ob.calculate(10)  # Prints "Total price is: 4000"
